Arquitetura do sistema

Topologia de Workflows

O sistema opera sobre uma arquitetura de orquestração hierárquica com dois workflows de entrada, um console multi-agentes central e quatro workflows de especialidade. A comunicação entre camadas é realizada via executeWorkflow nativo do n8n com passagem de contexto estruturado.

AGRO-E · TOPOLOGIA DE EXECUÇÃO
WF01 · 133 nós WF02 · 31 nós WF03-06 · 80 nós
Nota de arquitetura: O WF04 (Vencimentos — 48 nós) implementa pipeline completo de monitoramento documental com scoring automático 0–40 pts por documento. O WF06 (CRM/ERP — 11 nós) provê camada de integração para exportação de dados de oportunidades para sistemas externos. WF03 implementa escalada inteligente com handoff contextualizado para operador humano.
WF01 · ENTRADA + MONITOR PROATIVO
Gateway Multi-Canal
133 nós
Normalização universal de inputs (texto, áudio via Whisper, imagem via Vision, documento PDF). Roteamento por tipo de mensagem. Agregação de dados de 7 fontes simultâneas via SQL Postgres + Supabase + Open-Meteo + AwesomeAPI. Schedule trigger autônomo às 8h.
WF02 · CONSOLE MULTI-AGENTES
Orquestrador de Inteligência
31 nós
Agente Roteador (GPT-4.1-mini + Postgres Chat Memory) com 5 sub-agentes especializados, Think Tool × 2 para raciocínio explícito, Gmail Tool para envio de email, Calendar Tools × 3 para alertas e handoff para WF03 via encaminhar_responsavel_humano.
WF03 · ESCALADA HUMANA
Human-in-the-Loop
13 nós
Pipeline de escalada com preservação completa de contexto conversacional, priorização por criticidade e notificação multicanal do responsável humano designado.
WF04 · VENCIMENTOS
Monitoramento Documental
48 nós
Scoring automático de documentos agrícolas (0–40 pts). Análise de SLAs, licenças ambientais, certificados fitossanitários, ART, CAR. Alertas proativos com antecedência configurável.
WF05 · RELATÓRIO
Business Intelligence
8 nós
Geração e persistência de relatórios executivos no Supabase com score composto, ação recomendada, prazo, KPIs de impacto e histórico para análise de tendências.
WF06 · CRM/ERP
Integração Sistêmica
11 nós
Camada de integração para exportação de oportunidades e dados de clientes para CRMs e ERPs externos, habilitando operação em ecossistemas corporativos existentes.
Capacidades técnicas

Funcionalidades Implementadas

Cada capacidade foi validada em ambiente de produção durante a sessão de demonstração. Os diferenciais marcados com ★ representam features que elevam o sistema além do padrão de chatbots convencionais.

🎙️
Processamento de Áudio (ASR)
OPENAI WHISPER · EVOLUTION API · WAV/OGG DECODER
Download automatizado do áudio via Evolution API, decodificação do formato OGG/Opus e transcrição com OpenAI Whisper. O output textual entra no pipeline de NLU sem qualquer intervenção manual — o operador fala, o sistema compreende.
★ Diferencial — entrada multimodal
📄
Análise de Documentos (IDP)
PDF EXTRACT · CONTEXT INJECTION · SEMANTIC ANALYSIS
Extração de texto de PDFs e documentos recebidos via WhatsApp. O conteúdo é injetado no contexto de sessão e disponibilizado ao agente para análise semântica — contratos, laudos técnicos, licenças ambientais e certificados fitossanitários são processados inline.
★ Diferencial — IDP integrado
🖼️
Análise Visual (Computer Vision)
OPENAI VISION · IMAGE DECODE · MULTIMODAL CONTEXT
Imagens enviadas via WhatsApp (fotos de lavoura, documentos fotografados, registros de pragas) são analisadas pelo modelo Vision da OpenAI. A interpretação é incorporada à resposta agêntica como contexto semântico estruturado.
★ Diferencial — visão computacional
📱
Canal WhatsApp Bidirecional
EVOLUTION API · WEBHOOK · DEBOUNCE · DEDUP
Webhook persistente com normalização de payload multi-API (Evolution, UAZAPI, Z-API). Debounce de mensagens fracionadas via Redis. Deduplicação por event ID para garantia de entrega exactly-once. Latência P50 de 7s fim-a-fim.
Validado — 20+ execuções
⏰
Monitor Proativo Autônomo
SCHEDULE TRIGGER · 7 DATA SOURCES · ASYNC FANOUT
Schedule cron às 8h dispara coleta paralela de 7 fontes: PostgreSQL (operações/documentos), Supabase (clientes/catálogo/regiões/histórico/concorrentes), Open-Meteo via GPS e AwesomeAPI (câmbio USD/BRL). Zero intervenção humana.
Configurado — produção
🏗️
Motor de Inteligência Comercial
SQL JOINS · JS SCORING ENGINE · 4 OPPORTUNITY TYPES
Query SQL multi-tabela com COALESCE e json_agg retorna 5 datasets em uma única viagem ao banco. Engine JavaScript calcula 4 tipos de oportunidade: PRECO_FAVORAVEL (<-3%), EXPANSAO, RETENCAO (>120d), CROSS_SELL. Executa em <50ms.
R$1.56M identificados
📋
Agente Analista Documental
SCORING 0-40PTS · SLA TRACKING · RISK CLASSIFICATION
Sub-agente com scoring automático por documento (0–40 pts). Classifica: VENCIDO (score 40), CRÍTICO (<15d, score 35), ALTO (<30d, score 25), MÉDIO (<60d, score 15), BAIXO. SLA tracking com alertas proativos configuráveis por tipo de documento.
WF04 · 48 nós
🌧️
Agente Monitor Climático
OPEN-METEO API · GPS COORDINATES · THRESHOLD ENGINE
Consulta Open-Meteo API por coordenadas GPS de cada região da carteira. Thresholds configurados: precipitação >60mm → ALTO, >100mm → CRÍTICO; temp <5°C → risco de geada; seca >7 dias → ALTO. Output integrado ao score composto 0-100.
Open-Meteo por GPS
💱
Agente Analista Câmbio
AWESOMEAPI · USD/BRL · HEDGE RECOMMENDATION ENGINE
Polling AwesomeAPI para USD/BRL em tempo real. Thresholds: variação >1.5% → ALERTA, >2.5% → CRÍTICO; tendência de alta por >5 dias consecutivos → recomendação de hedge. Impacto cambial integrado às propostas de insumos importados.
AwesomeAPI USD/BRL
📧
Entrega de Email (MTA)
GMAIL API · OAUTH2 · TOOL CALLING · ASYNC DELIVERY
Gmail Tool integrada ao agente via OAuth2 com escopo gmail.send. Dispatched via tool calling do GPT-4.1-mini — o modelo decide autonomamente quando disparar o email com base no contexto da conversa. Entrega confirmada na inbox do operador.
Entregue — confirmado
📅
Gestão de Alertas (Calendar)
GOOGLE CALENDAR API · OAUTH2 · CREATE/UPDATE/DELETE
Três Calendar Tools disponíveis ao agente: create_event, update_event e cancel_event. Permite criar alertas de vencimentos documentais, lembretes de visitas a clientes e marcos de operação diretamente da conversa WhatsApp.
3 tools · configurado
🧠
Memória Conversacional Persistente
POSTGRES CHAT MEMORY · SESSION SCOPING · CONTEXT WINDOW
Postgres Chat Memory com escopo por número de telefone. Histórico de conversas armazenado em agroe_contexto com perfil do operador (nome, email, preferências). O agente reconhece contexto de sessões anteriores sem re-identificação.
Persistência cross-session
📊
Relatórios de Gestão (BI Layer)
SUPABASE PERSIST · RISK SCORE · IMPACT MODELING
Cada ciclo de análise persiste registro estruturado em agroe_relatorios_gestao: tipo de risco, score 0-100, ação recomendada, deadline, receita potencial em reais e histórico de execuções. Dados disponíveis para BI externo via API REST Supabase.
REST API disponível
👤
Onboarding Zero-Touch
AUTO-REGISTER · SUPABASE INSERT · ai_service=active
Números não cadastrados são provisionados automaticamente em agroe_contexto com ai_service='active' na primeira interação. Sem fluxo de cadastro manual — o sistema está operacional para qualquer novo operador desde a primeira mensagem recebida.
Onboarding zero-touch
🔁
Human-in-the-Loop (HITL)
WF03 ESCALADA · CONTEXT HANDOFF · PRIORITY ROUTING
Tool encaminhar_responsavel_humano aciona WF03 com contexto completo da sessão. Gatilhos: embargo documental, produtor irresponsivo >10 dias, score de risco >85, ou solicitação explícita do operador. Preserva histórico integral para continuidade pelo humano.

Operação autônoma

Pipeline Proativo — Schedule 08:00

O sistema opera como um analista permanente. Sem qualquer interação do operador, o WF01 dispara às 8h e conduz um ciclo completo de coleta, análise e entrega do relatório executivo diário.

08:00:00 — CRON TRIGGER
Disparo Autônomo — Monitor Proativo 8h
n8n aciona a branch proativa do WF01 via Schedule Trigger. Nenhuma intervenção humana necessária. O sistema inicia fan-out paralelo de 7 coletas de dados.
08:00–08:03 — DATA COLLECTION
Coleta Paralela de 7 Fontes
PostgreSQL (documentos/operações/histórico), Supabase (clientes/catálogo/regiões/concorrentes), Open-Meteo GPS (dados climáticos por região da carteira), AwesomeAPI (câmbio USD/BRL em tempo real).
08:03–08:05 — SCORING ENGINE
Motor Agro-e — Score Composto 0–100
Engine JavaScript consolida os 7 datasets. Calcula score composto de risco (documental + climático + cambial + histórico de inadimplência) e identifica oportunidades comerciais por tipo. Output estruturado para o agente.
08:05–08:15 — AI ORCHESTRATION
WF02 — Console Multi-Agentes (GPT-4.1-mini)
Agente Roteador aciona sub-agentes por domínio. Think Tool × 2 garante raciocínio explícito e auditável. Relatório executivo gerado com alertas classificados por criticidade e oportunidades ordenadas por receita potencial.
08:15 — DELIVERY
Entrega Multi-Canal + Persistência
Análise completa entregue via WhatsApp ao operador. Email automático disparado via Gmail OAuth2. Relatório persistido em agroe_relatorios_gestao (Supabase) para rastreabilidade histórica e BI.
Evidência de funcionamento autônomo: A caixa de entrada do operador exibe emails com padrão "agro-e Gestão | X crítico(s) · Y alto(s) · Z op..." de dias anteriores à sessão de demonstração — prova de que o sistema opera de forma contínua e autônoma, independentemente de qualquer interação no WhatsApp.
