Os três pilares
🛡️
Gestão de Riscos
Monitora documentos vencidos, riscos climáticos por GPS e exposição cambial USD/BRL. Score composto 0–100 com 5 fatores ponderados. Classifica: CRÍTICO · ALTO · MÉDIO · BAIXO. Alerta proativo antes que o problema vire perda.
Zero surpresas documentais — alertas com 30 dias de antecedência
📈
Inteligência Comercial
Cruza histórico de compras com variação de preços, clima e mapeamento de concorrência. Identifica 4 tipos de oportunidade: preço favorável, antecipação climática, expansão estratégica e risco de churn. Saída acionável por cliente.
Oportunidades identificadas automaticamente — diariamente, por cliente
📊
Gestão e Rastreabilidade
Cada alerta vira item rastreável com status, responsável e histórico completo de mudanças: pendente → em andamento → resolvido. Relatório executivo diário via email HTML e flash WhatsApp. Audit trail completo (LGPD).
Total visibilidade para gestores tomarem decisões com dados

Arquitetura do sistema

6 Workflows · 244 Nós

Orquestração hierárquica com dois workflows de entrada, um console multi-agentes central e quatro workflows de especialidade. Comunicação entre camadas via executeWorkflow com passagem de contexto estruturado.

WF 01 · ENTRADA + MONITOR PROATIVO
Gateway Multi-Canal
133 nós
Normalizador Universal (Evolution/UAZAPI/ZAPI), extração de texto de áudio via Whisper, visão computacional (OpenAI Vision), extração de PDFs. Modo proativo: 7 fontes simultâneas — PostgreSQL, Supabase, Open-Meteo (GPS) e AwesomeAPI. Sanitizador de resposta que bloqueia JSON/erros antes do envio ao WhatsApp.
WF 02 · CONSOLE MULTI-AGENTES
O Cérebro — Orquestrador de IA
31 nós
Agente Roteador (GPT-4.1-mini + Postgres Chat Memory) com 5 sub-agentes especializados. Think Tool × 2 para raciocínio explícito e auditável. Gmail Tool (OAuth2), Calendar Tools × 3, encaminhar_responsavel_humano → WF03. Temperatura calibrada por função em cada sub-agente.
WF 03 · ESCALADA HUMANA
Human-in-the-Loop
13 nós
Ativado quando score > 85 ou decisão que exige humano. Pausa IA (ai_service=pause), notifica responsável com contexto completo. Retomada automática quando status voltar para active. Compatível com Evolution/UAZAPI/ZAPI.
WF 04 · MONITOR DE VENCIMENTOS
Monitoramento Documental
48 nós
Executa 2× por dia: 7h (urgentes: vencidos ou ≤2 dias) e 12h (próximos: 3–14 dias). Scoring 0–40pts por documento. Gera mensagem WhatsApp humanizada por fazenda com nome, documento, órgão e prazo.
WF 05 · RELATÓRIO DE GESTÃO
Business Intelligence
8 nós
Às 9h: email HTML executivo com cards de resumo (críticos/altos/oportunidades/SLA), cartões detalhados e tabela completa + flash WhatsApp de 5 linhas. Rastreabilidade: pendente → em_andamento → resolvido.
WF 06 · INTEGRAÇÃO CRM/ERP
Adaptador Universal
11 nós
Conecta o agro-e a qualquer CRM/ERP via schema padronizado. 5 conectores pré-configurados: Salesforce, HubSpot, Siagri, TOTVS Agri, AgroSmart + webhook genérico ativo. Ativação: só configurar credencial nas variáveis n8n. Zero dev adicional.
Arquitetura de memória — 3 camadas
Redis · TTL 1h
Curto Prazo
Contexto ativo da sessão corrente. Debounce de mensagens fracionadas. Idempotency keys para garantia exactly-once.
PostgreSQL · 90 dias
Médio Prazo
Histórico conversacional por operador (Postgres Chat Memory). Recuperado antes de cada análise para contextualização precisa.
Supabase · Permanente
Longo Prazo
Perfil do produtor, resumo de contexto, histórico de execuções, relatórios de gestão e audit trail LGPD.

Processamento de Áudio (ASR)
OPENAI WHISPER · EVOLUTION API · OGG/OPUS DECODER
Download do áudio via Evolution API, decodificação OGG/Opus e transcrição com Whisper. Output textual entra no pipeline de NLU sem intervenção manual — o operador fala, o sistema compreende e responde.

Análise de Documentos (IDP)
PDF EXTRACT · CONTEXT INJECTION · SEMANTIC ANALYSIS
Extração de texto de PDFs recebidos via WhatsApp. Conteúdo injetado no contexto de sessão e disponibilizado ao agente — contratos, laudos técnicos, licenças ambientais e certificados fitossanitários processados inline.
★ Diferencial — IDP integrado

Visão Computacional
OPENAI VISION · MULTIMODAL CONTEXT · IMAGE DECODE
Imagens (fotos de lavoura, pragas, documentos fotografados) analisadas pelo modelo Vision. A interpretação é incorporada à resposta agêntica como contexto semântico estruturado.
★ Diferencial — Computer Vision

WhatsApp Bidirecional
EVOLUTION API · WEBHOOK · REDIS DEBOUNCE · DEDUP
Webhook persistente com normalização multi-API (Evolution/UAZAPI/ZAPI). Debounce via Redis + deduplicação por event ID para exactly-once delivery. Latência P50: 7s fim-a-fim.
Validado · 20+ execuções

Monitor Proativo 8h
CRON TRIGGER · 7 FONTES · ASYNC FAN-OUT
Cron diário (seg–sáb) dispara fan-out paralelo: PostgreSQL, Supabase (5 tabelas), Open-Meteo por GPS, AwesomeAPI USD/BRL. Zero intervenção humana. Agente decide subagentes, ordem e canais de saída autonomamente.
Autônomo · Produção
🏗️
Motor de Inteligência Comercial
SQL MULTI-TABLE · JS SCORING · 4 TIPOS OPP
Query SQL com COALESCE + json_agg retorna 5 datasets em 1 roundtrip ao banco. Engine JS calcula: Preço Favorável (<−3%), Antecipação Climática, Expansão Estratégica, Risco de Churn. Execução <50ms.

Agente Analista Documental
SCORING 0-40PTS · SLA TRACKING · WF04
Score por documento: Vencido=40, ≤7d=35, ≤30d=25, ≤60d=15, >60d=0. Classifica CRÍTICO/ALTO/MÉDIO/BAIXO. SLA tracking com alertas proativos 2× dia.
WF04 · 48 nós
🌧️
Agente Monitor Climático
OPEN-METEO · GPS COORDS · THRESHOLD ENGINE
Consulta Open-Meteo por coordenadas GPS de cada região. Thresholds: >60mm → ALTO, >100mm → CRÍTICO, temp <5°C → geada, seca >7d → ALTO. Score contribui no composto 0–100.
Open-Meteo por GPS
💱
Agente Analista Câmbio
AWESOMEAPI · USD/BRL · HEDGE ENGINE
Polling AwesomeAPI em tempo real. Thresholds: >1.5% → ALERTA, >2.5% → CRÍTICO, alta >5 dias → hedge recomendado. Impacto integrado às propostas de insumos importados.
AwesomeAPI real-time

Entregue · Confirmado

3 tools · Configurado
